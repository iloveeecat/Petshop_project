# Проекты

## Pet-проект — Интернет-магазин животных (приют)

### Функционал
- **Каталог и карточки животных:** просмотр категорий и карточек с фото, описанием и ценой/стоимостью содержания.  
- **Фильтрация по категориям:** загрузка данных по `categoryId`.  
- **Админ‑панель категорий:** создание, редактирование и удаление категорий с валидацией и мгновенным обновлением списка.  
- **Админ‑панель животных:** создание, редактирование и удаление карточек животных с клиентской валидацией и блокировкой кнопок во время сохранения. 
- **Клиентская валидация форм:** пустые поля, неверные URL/картинки, отрицательные значения.

### Технологии
React, Redux, Zustand, React Router, TanStack Query (React Query), Axios, React Hook Form + Yup, Tailwind CSS, Docker

- **Redux Toolkit:** управление сложным клиентским состоянием карточек животных.  
- **Async thunks** (`fetchProducts`, `addProduct`, `editProduct`, `removeProduct`) упростили работу с асинхронными запросами и синхронизацию UI с API.  
- **Zustand:** рассматривался как альтернатива Redux для менее сложных сценариев. В проекте делала акцент на сравнении подходов и оставила Redux для более сложного CRUD.  
- **React Router:** для SPA-навигации.  
- **TanStack Query (React Query):** серверное состояние, автоматическое кеширование, обновление UI после мутаций через `invalidateQueries`, индикаторы `isLoading` и `isPending`.  
- **Axios:** сетевой слой API (`categoriesApi.ts`, `productsApi.ts`).  
- **React Hook Form + Yup:** декларативная валидация форм, UX: подсветка полей, сообщения об ошибках под конкретным input, минимальные ререндеры.  
- **Docker:** контейнеризация приложения, сборка React → Nginx, корректная работа SPA‑роутинга (редирект всех неизвестных путей на `index.html`).  

### Архитектура и паттерны
- Feature-based архитектура (модули `Admin/Animals`, `Admin/Categories`)  
- Singleton-сервисы (`axios`-инстансы, Redux store, QueryClient)  
- Фабричные функции для API и Redux thunks  
- Observer-подход через React Query / Redux (подписка компонентов на изменение состояния)  
- Разделение контейнерных и презентационных компонентов  
- Схемы валидации (`Yup`) как отдельный слой правил  

### Проблемы и решения
1. **Обновление списков после CRUD без перезагрузки:**  
   После создания/редактирования/удаления категория или животное не появлялись в списке до ручного обновления страницы.  
   **Решение:** перевела операции на `useMutation` (React Query) + `queryClient.invalidateQueries` для соответствующих `queryKey`.

2. **Совмещение Redux Toolkit и React Query:**  
   Нужно было решить, где хранить данные: всё в Redux, всё в React Query или разделить ответственность.  
   **Решение:** серверное состояние публичных списков категорий и продуктов — через React Query, локальное сложное состояние — через Redux Toolkit.
